# Phase 1 — UI Design Contract

> Visual and interaction contract for the Soft Gate screen + foundational design tokens that every subsequent phase inherits. Generated by gsd-ui-researcher; verified by gsd-ui-checker.

This UI-SPEC has TWO layers:

1. **Foundation tokens** — locked once here, consumed by Phases 1, 3, 5, 6, 7. Color palette, typography, spacing, radii, shadows, motion. Future phases extend, never redefine.
2. **Gate screen contract** — the only surface rendered in Phase 1. Composition, states, micro-interactions, copywriting, a11y.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (greenfield, hand-rolled tokens — no shadcn/Radix/component library) |
| Preset | not applicable |
| Component library | none — single bespoke `<form>` with two controls |
| Icon library | none in Phase 1 (gate is "carta + candela + buio", no icons). Future phases (3+) may add `lucide-vue-next` for A−/A+ controls only — flagged here, NOT installed in Phase 1. |
| Style approach | Vanilla CSS + CSS Custom Properties on `:root` (`tokens.css`). Imported once from `src/main.ts`. No UnoCSS / Tailwind in Phase 1 — keep bundle minimal until Phase 3 introduces real layout needs. |
| Font loading | Self-hosted via `@fontsource/cormorant-garamond` (serif) + system-ui sans stack. WOFF2 only. `font-display: swap`. `latin-ext` subset for Italian accents (`àèéìòù ÀÈÉÌÒÙ «» — …`). |
| Fonts | **Serif**: Cormorant Garamond (provisional v1 lock — re-confirmable at Phase 3 render-test against EB Garamond / Lora). **Sans**: system-ui, -apple-system, "Segoe UI", Roboto, sans-serif (zero KB, instant render, used only for the `Entra` button label and aria-live error text). |

**Justification for Cormorant Garamond:** transitional Italian serif with high contrast and humanist warmth — matches Han Kang / scrittoio antico / pre-war letter register. Carries Italian accents natively. Free, OFL-licensed, full Latin Extended.

## Spacing Scale

Declared values (multiples of 4, exposed as `rem` for A−/A+ compatibility from Phase 3):

| Token | Value (px @ 16px root) | rem | Usage |
|-------|------------------------|-----|-------|
| `--sp-xs` | 4px | 0.25rem | Hairline gaps, paper-strip vertical padding |
| `--sp-sm` | 8px | 0.5rem | Inline label-to-control gaps |
| `--sp-md` | 16px | 1rem | Default element spacing, button padding-x |
| `--sp-lg` | 24px | 1.5rem | Gap candela ↔ campo password |
| `--sp-xl` | 32px | 2rem | Gap campo ↔ bottone Entra |
| `--sp-2xl` | 48px | 3rem | Vertical padding on viewport edges (mobile) |
| `--sp-3xl` | 64px | 4rem | Vertical padding on viewport edges (desktop) |

**Exceptions:** Touch targets — input field and `Entra` button must be ≥ 44px tall regardless of font scale. Implemented via `min-block-size: 44px`, NOT padding hacks (zoom-safe).

**Forward signal:** Phase 3+ will add `--sp-3xs: 2px` (only) for polaroid border-on-shadow seam — not declared here to keep the scale honest.

## Typography

Exactly **3 sizes, 2 weights** in Phase 1. Sizes are fluid via `clamp()` so the layout reflows under browser zoom up to 200% without horizontal overflow.

| Role | Size | Weight | Line Height | Family | Usage |
|------|------|--------|-------------|--------|-------|
| Body | `clamp(16px, 1rem + 0.2vw, 18px)` | 400 (regular) | 1.5 | Cormorant Garamond | Password input value (the ink on the paper strip) |
| Label | `clamp(13px, 0.825rem + 0.1vw, 14px)` | 400 | 1.4 | system-ui sans | Error message under the strip ("password non corretta") + button label "Entra" |
| Display | not used in Phase 1 | — | — | — | Reserved for poem titles in Phase 3 |

**Letter-spacing:**
- Body (serif): default (0)
- Label (sans, button "Entra"): `0.04em` (slight tracking — the only "design tic" allowed; reads as quiet typography, not loud UI)

**Italic:** reserved for poem dates (Phase 3). Not used in Phase 1.

**Forward signal (locked here for Phases 3+):**
- Poem body: `clamp(18px, 1.125rem + 0.25vw, 22px)`, weight 400, line-height 1.65, Cormorant Garamond, max-width 34ch
- Poem title: `clamp(24px, 1.5rem + 0.5vw, 32px)`, weight 500, line-height 1.2, Cormorant Garamond
- Polaroid `aria-label` is screen-reader-only; no visible label typography needed

`hyphens: auto` + `lang="it"` set on `<html>` from Phase 1 (cheap, non-visible in Phase 1, eliminates retrofit risk in Phase 3).

| Role | Token(s) | Coverage | Usage |
|------|----------|----------|-------|
| Dominant (60%) | `--c-soot-800` + `--c-soot-900` (radial gradient) | Background of entire viewport | Soffitta scura |

### Accent reserved for (explicit list — not "all interactive elements")

- **`--c-focus`**: visible focus ring on the password field AND the Entra button. Nothing else.
- **`--c-ink-900`**: only the typed password value on the paper strip + the "Entra" button label.
- **`--c-error`**: only the failure message under the paper strip. Never used for borders, never for the field's focus ring.
- **`--c-amber-500/300`**: NOT visible in Phase 1. Reserved for Phase 5 candela accesa. Declared here so Phase 5 doesn't reinvent.

### Background composition (Phase 1 gate)

```css
background:
  radial-gradient(
    ellipse 70% 50% at 50% 35%,
    var(--c-soot-700) 0%,
    var(--c-soot-800) 55%,
    var(--c-soot-900) 100%
  );
/* + a low-alpha SVG noise overlay (~ 800-byte inline SVG, ~5% opacity) for "carta/legno" texture without bitmap weight */
```

**No bitmap textures in Phase 1.** Soffitta texture is pure CSS gradient + inline SVG noise (target < 2 KB). This keeps FCP fast and avoids decoding cost during gate render.

## Motion Tokens

All durations and easings declared once. Reduced-motion equivalents declared in parallel — never bolted on.

| Token | Default value | `prefers-reduced-motion: reduce` value |
|-------|---------------|----------------------------------------|
| `--motion-duration-instant` | 0ms | 0ms |
| `--motion-duration-fast` | 150ms | 0ms |
| `--motion-duration-base` | 280ms | 0ms |
| `--motion-duration-slow` | 400ms | 0ms |
| `--motion-duration-deliberate` | 800ms (artificial submit delay — see Failure UX) | 800ms (kept — accessibility-orthogonal, masks PBKDF2 latency) |
| `--motion-ease-out` | `cubic-bezier(0.16, 1, 0.3, 1)` | linear |
| `--motion-ease-soft` | `cubic-bezier(0.4, 0, 0.2, 1)` | linear |
| `--motion-shake-amplitude` | 4px | 0 |

**Rule:** any animated property must consult `@media (prefers-reduced-motion: reduce)` and substitute the reduced value. This is enforced at the component level via the reactive `motion: 'full' | 'reduced'` flag (RESEARCH.md Reconciliation 3).

## Gate Screen — Layout & Composition

### Viewport composition (mobile-first, scales up cleanly)

```
┌────────────────────────────────────────┐
│                                        │
│            (vertical breath)           │
│                                        │
│             ╔═══════╗                  │
│             ║   ▓   ║   ← candela       │
│             ║   ▓   ║     (spenta)      │
│             ║ ║▓▓▓║ ║                  │
│             ║ ║▓▓▓║ ║                  │
│             ╚═══════╝                  │
│                                        │
│           gap = --sp-lg (24px)         │
│                                        │
│      ──────────────────────────        │
│       (paper strip, ink underline)     │ ← input
│      ──────────────────────────        │
│                                        │
│           gap = --sp-xl (32px)         │
│                                        │
│              [  Entra  ]               │ ← button
│                                        │
│                                        │
│            (vertical breath)           │
│                                        │
└────────────────────────────────────────┘
```

### Centering strategy

- Outer container: `min-height: 100dvh` (NOT `100vh` — iOS Safari trap, Pitfall 8). Fallback `min-height: 100vh` declared first.
- Display: `grid; place-items: center;` — single center group holds candle + form.
- Center group: `display: flex; flex-direction: column; align-items: center;`
- Center group max-width: `min(420px, 100% - var(--sp-xl) * 2)` so the layout never butts against the viewport edge.

### Candela (spenta)

- **Implementation:** inline SVG (~ 1.2 KB), NOT an image asset. Embedded in the Vue SFC's `<template>` so it ships in the gate's initial render — no network round-trip.
- **Dimensions:** `width: clamp(56px, 5vw + 40px, 88px)`. Aspect maintained.
- **Colors:** wax body in `--c-paper-200` with subtle vertical gradient toward `--c-paper-100` at top. Wick in `--c-soot-900` (un-lit char tip). NO flame. NO glow halo.
- **Drop shadow:** `--shadow-candle-cold`.
- **Position:** centered horizontally; bottom edge sits exactly `--sp-lg` (24px) above the paper strip's top edge.
- **State during submit:** very subtle wick "tremolio" — see Micro-interactions below. Default state: utterly still.

### Campo password — la "riga su un pezzo di carta"

Visual metaphor: a torn strip of aged ivory paper, lit by an absent candle, with the password written in ink along an underline.

- **Strip dimensions:** `width: clamp(220px, 60vw, 340px); min-height: 44px; padding: var(--sp-sm) var(--sp-md);`
- **Strip background:** `--c-paper-100` with a 0.5px subtle inner shadow at top and bottom (`inset 0 1px 0 rgba(0,0,0,0.06), inset 0 -1px 0 rgba(0,0,0,0.06)`) — sells "thickness of paper".
- **Strip edges:** `border-radius: 0` (the paper is torn, not die-cut). Optional: a subtle clip-path with 1–2px irregular bumps on left/right edges for "torn" feel — keep tasteful, NOT zigzag.
- **Strip drop shadow:** `--shadow-paper`.
- **The input itself:** native `<input type="password">` — required for password manager support and platform autofill.
  - `background: transparent; border: 0; outline: 0; color: var(--c-ink-900); font: 400 var(--fs-body)/1.5 "Cormorant Garamond", serif; letter-spacing: normal; caret-color: var(--c-ink-700);`
  - Underline ink: `border-bottom: 1px solid var(--c-ink-700);` rendered with subtle `border-style` solid (NOT dashed). When focused, underline transitions to `2px` and `--c-ink-900` over `--motion-duration-fast`. Reduced-motion: instant swap.
  - Width: 100% of strip's content box.
- **Placeholder:** the literal string `password` in lowercase, color `--c-ink-700` at 50% alpha (`color: rgba(58, 44, 28, 0.5)` — verified contrast 4.7:1 against `--c-paper-100`, AA for placeholder which is decorative-only; the field's accessible name comes from `aria-label="password"`, NOT the placeholder).
- **Autocomplete:** `autocomplete="current-password"` — lets password managers do their job. Per soft-gate design, this is intentional.
- **`spellcheck="false"` + `autocapitalize="off"` + `autocorrect="off"`** — phrase passwords with spaces ("memento mori") must not be auto-capitalized on iOS.
- **`inputmode` not set** — let platform default decide (avoids forcing numeric keyboard).
- **No label DOM node** — but `aria-label="password"` (Italian) on the input. Per CONTEXT.md (no titolo, no benvenuto).

### Bottone "Entra"

- **Width:** `width: clamp(220px, 60vw, 340px);` — matches strip width for axis alignment.
- **Min-height:** `44px` (touch target).
- **Padding:** `var(--sp-sm) var(--sp-md)`.
- **Background:** `--c-paper-200` (slightly darker than the strip — reads as "another piece of paper, slightly older"). On hover (desktop only): `--c-paper-100`. Hover effect MUST also have a non-hover analog — color change is supplementary, not primary feedback.
- **Color:** `--c-ink-700`. Font: `400 var(--fs-label)/1.4 system-ui, sans-serif; letter-spacing: 0.04em;`
- **Border:** `1px solid rgba(58, 44, 28, 0.25)`.
- **Border-radius:** `--radius-sm` (2px).
- **Shadow:** `--shadow-paper` (matches the strip's lift).
- **`:focus-visible`:** `outline: 2px solid var(--c-focus); outline-offset: 3px;` — visible on dark soffitta, AA contrast 11.6:1.
- **Disabled state during submit (800ms):** `cursor: progress; opacity: 0.6;` + `aria-disabled="true"` (but NOT `disabled` attribute, so the button stays in tab order). Label remains "Entra" — does NOT change to "Verifica..." (microcopy locked to one word).

## States (gate)

### 1. Rest

Soffitta dark, candle still and dark, paper strip pristine, no error message rendered (error region is `aria-live` but empty).

**Initial focus:** the password input is autofocused on mount via `ref.focus()` in `onMounted`. (Autofocus is generally discouraged on landing pages; here it's appropriate because the gate is a single-purpose modal-like surface — same exception that login screens take.)

### 2. Focus (campo)

- Strip's bottom underline transitions from 1px `--c-ink-700` to 2px `--c-ink-900` (`--motion-duration-fast`).
- A `--c-focus` outline ring appears AROUND the strip (not the input directly), `outline: 2px solid var(--c-focus); outline-offset: 4px;`. Visible on dark soffitta.
- Caret blinks at default rate, color `--c-ink-700`.
- Reduced-motion: underline swap instant; outline appears instantly.

### 3. Submitting (during PBKDF2 + 800ms artificial delay)

- Button: `cursor: progress; opacity: 0.6; aria-disabled="true";` Label unchanged.
- Input: stays focused, value visible.
- **Candle micro-feedback (NOT a spinner):** the wick tip pulses very softly — a 1–2px vertical opacity oscillation on the char tip, ~1.5s cycle, max ±15% opacity. NEVER becomes flame, NEVER changes color, NEVER glows. It's "the candle is thinking, not lit."
  - Reduced-motion: NO pulse. The button's aria-disabled state is the only feedback. Screen-reader announcement (see a11y) covers the rest.
- No spinner. No progress bar. No skeleton.

### 4. Error

- Strip "vibra" (shake): `transform: translateX()` keyframe, 4 keyframes at -4px, +4px, -2px, 0 over `--motion-duration-base` (280ms), `ease-out`.
  - Reduced-motion: NO shake. Strip is static.
- Error region renders the literal text: `password non corretta` — `<p role="status" aria-live="polite">`, font `var(--fs-label)`, color `--c-error`, italic, positioned `--sp-sm` (8px) under the strip, centered.
- The input value is auto-selected (`input.select()`) so the user can immediately retype/overtype without first clearing.
- Focus REMAINS on the input. Never jumps to the error message.
- Once the user types again (input event), the error message is removed from the DOM (clean slate). It does NOT linger.

### 5. Success-fading (the unlock crossfade)

- The entire gate view crossfades to opacity 0 over `--motion-duration-slow` (400ms), `--motion-ease-soft`. Concurrently, the post-unlock placeholder view crossfades in from opacity 0 to 1 over the same duration.
- During crossfade, both views are mounted. Pointer events disabled on the outgoing gate (`pointer-events: none`).
- After fade completes, the gate component is unmounted (Vue Router transition).
- Reduced-motion: instant swap. No crossfade. The placeholder view appears in one frame.
- The candle stays SPENTA throughout the crossfade in Phase 1. (POLI-01 / Phase 5+ will choreograph the lighting, separately.)

## Micro-interactions & Motion (Phase 1)

| Interaction | Default | Reduced-motion |
|-------------|---------|----------------|
| Caret blink in input | Browser default (no override) | Browser default |
| Field focus underline | 150ms ease-out, 1px → 2px, ink-700 → ink-900 | Instant swap |
| Field focus ring (outline) | Instant appear (no fade) | Instant appear |
| Button hover (desktop only) | Background `--c-paper-200` → `--c-paper-100`, 150ms | Instant swap (color change still happens — non-motion) |
| Submit-pending wick pulse | ~1.5s sine-pulse, ±15% opacity on wick tip | NONE — stays static |
| Submit-pending button | `opacity: 1 → 0.6`, instant + cursor change | Same (opacity change is not "motion" — it's a state) |
| Error shake | 280ms, 4-keyframe translateX, ease-out, max amplitude 4px | NONE — strip stays still |
| Error message reveal | Appears instantly in DOM (no fade — `aria-live` requires immediate DOM presence) | Same |
| Unlock crossfade | 400ms opacity fade both ways | Instant cut |

**Rule:** `prefers-reduced-motion: reduce` users get full functional parity. State changes are still communicated — via cursor changes, `aria-disabled`, and `aria-live` text updates — but no transforms or transitions on visual properties (except opacity changes that signal *state*, which are kept).

## Accessibility Contract (Phase 1)

| Requirement | Implementation |
|-------------|----------------|
| Touch targets ≥ 44×44px | Input strip: `min-height: 44px`. Button: `min-height: 44px`. |

| Focus visible | `:focus-visible` always renders an outline using `--c-focus` (AA contrast 11.6:1 against the soffitta background). Mouse-only focus does NOT show the ring (intentional — `:focus-visible`, not `:focus`). |

(7) Success: route changes to `/`, the unlocked main region announces (Phase 3+ adds proper landmark). |
| `prefers-reduced-motion` | Honored everywhere — see Motion table. |
| Browser zoom 200% | Layout MUST NOT introduce horizontal scroll. Verified via `clamp()` widths and `min(420px, 100% - X)` constraints. |
| Color contrast (AA) | All combinations verified above. AAA on all text-on-paper pairs; AA on error and on focus ring. |
| Form semantics | A real `<form>` element with `@submit.prevent` Vue handler. Submit triggered by Enter or button click. No `<div onClick>` masquerading as a button. |

## Responsive Behavior

| Breakpoint | Behavior |
|------------|----------|
| Mobile portrait (≤ 480px) | Vertical layout. Padding-x: `var(--sp-md)` (16px). Candle: ~56px. Strip width: 60vw, min 220px. Generous `--sp-2xl` (48px) vertical breathing room above and below center group. No hover effects (touch-first). |
| Mobile landscape | Same vertical layout, candle scales down a touch via `clamp()`. Vertical breathing tightens to `--sp-lg`. |
| Tablet (481–1024px) | Same vertical layout. Candle: ~72px. Strip width: 50vw, min 280px. Padding: `--sp-2xl`. |
| Desktop (≥ 1025px) | Same vertical layout. Candle: ~88px. Strip width: 340px (clamp ceiling). Padding: `--sp-3xl`. Hover effects on the button enabled. |

**Single layout, fluid scaling.** No layout swap between breakpoints — only `clamp()`-driven size changes. This eliminates layout shift across breakpoints.

`100dvh` for outer container, with `100vh` fallback declared first (iOS Safari Pitfall 8 mitigation).

## Implementation Notes for the Planner

1. **Single CSS entry point:** `src/styles/tokens.css` declares all `--c-*`, `--sp-*`, `--fs-*`, `--motion-*`, `--shadow-*`, `--radius-*` custom properties on `:root`. Imported once from `src/main.ts`. No second file needed in Phase 1.
2. **Gate component file:** `src/views/GateView.vue` (or `src/gate/GateView.vue` per CONTEXT.md D-discretion) — a single SFC containing the form, the inline SVG candle, and the scoped styles consuming the tokens.
3. **Font setup:** `npm install -D @fontsource/cormorant-garamond` then `import '@fontsource/cormorant-garamond/400.css'; import '@fontsource/cormorant-garamond/latin-ext-400.css';` from `main.ts`. Font weight 400 only in Phase 1 (500 reserved for Phase 3 poem titles — install when needed, not now).
4. **Inline SVG candle:** keep it in the SFC, not a separate file. ~1.2 KB.
5. **The 800ms artificial delay** is implemented in the gate composable (`useGate.ts`), NOT in the component. The composable runs PBKDF2 then `await Promise.all([pbkdf2Result, sleep(800ms)])` — `Promise.all` over the longer of the two ensures the visible delay is at least 800ms regardless of PBKDF2 actual timing.
6. **Crossfade implementation:** Vue Router's `<router-view v-slot>` + `<Transition name="crossfade" mode="out-in">` with `--motion-duration-slow`. The reduced-motion variant disables the transition class via `:disabled` on `<Transition>` bound to `useReducedMotion()`.
7. **No GSAP in Phase 1.** GSAP is locked in the stack but only imported starting Phase 5/6. Keep Phase 1 bundle minimal.
8. **No icons in Phase 1.** Don't install `lucide-vue-next` yet.
